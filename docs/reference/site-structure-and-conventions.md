# Site structure and content conventions

## Content lanes

- **Notes (`/notes/`)**: long-form writeups.
- **How-to (`/how-to/`)**: task-oriented procedures and checklists.
- **Reference (`/reference/`)**: stable definitions and conventions.
- **Policies (`/policies/`)**: normative operating contracts.
- **Prompt blocks (`/prompts/`)**: copy/paste prompts aligned to policies.

## Linking convention
Use plain Markdown links (no Liquid) to keep links readable in both GitHub UI and built Pages.
